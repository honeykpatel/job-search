- Backend error chain:

  - Backend import fails before app is served.
  - Import path:
    - `app.api`
    - imports `collectors.rss_remoteok`
    - that module imports `feedparser`

- Meaning:
  - Frontend proxy is not the root problem.
  - Frontend cannot connect because backend process dies during import.

4. Root cause confirmed in this session
- `frontend/vite.config.js` proxies `/api` to `http://127.0.0.1:8000`.
- That target matches the backend host/port shown in the user logs.
- Therefore proxy config is not the immediate issue.
- Immediate blocker is missing Python dependency in the environment used to run Uvicorn.
- Repo manifests do list `feedparser`, but the active Python installation used by the user did not have it available at runtime.
- Likely interpretation:
  - user ran system Python instead of a prepared virtual environment, or
  - dependencies were not installed in the interpreter currently used for `python -m uvicorn ...`
